## 1. 产品简介

**Ebbinghaus Memo** 是一款基于艾宾浩斯遗忘曲线的多语言背单词应用。

**核心特性：**

- **6 阶段间隔重复算法（SRS）**：按照 `[1, 2, 4, 7, 15, 30]` 天的科学节奏自动排期复习
- **AI 自动生成释义**：添加新词时由 AI 自动生成中文释义、双语例句、艾宾浩斯助记
- **5 种目标语言词库**：英语 / 日语 / 西班牙语 / 法语 / 葡萄牙语
- **6 种 UI 界面语言**：中文 + 5 种目标语言任意切换
- **错词循环重考**：复习中答错的词会在同一次会话内立即重考，直到记住
- **勋章成就系统**：12 枚勋章覆盖"学习坚持 / 词库探索 / 终极掌握"三大维度
- **真人发音 + 浏览器 TTS 兜底**：每个单词都能听到标准发音

**开始使用：**

打开浏览器访问应用地址（本地开发环境默认为 `http://localhost:3003`），注册账号即可开始。

## 2. 账号注册与登录

### 2.1 注册新账号

1. 在登录页顶部点击 **"邮箱注册"** Tab。
2. 填写以下信息：
   - **电子邮箱**：作为账号唯一标识，登录时使用
   - **密码**：至少 6 位
   - **个性昵称**：导航栏和个人资料中显示的名称
   - **每日复习目标（词）**：5–200 之间，默认值 **15**
3. 点击 **"完成注册并生成词库"**。
4. 注册成功后会自动登录并跳转到仪表盘。

> 💡 **关于"每日复习目标"**：用于系统统计与提醒，建议初学者保持默认 15，熟练后可到个人资料页修改。

### 2.2 登录已有账号

1. 在登录页保持 **"登录账号"** Tab。
2. 输入邮箱与密码。
3. 点击 **"登录智能词库"**。

### 2.3 忘记密码？

应用**不提供邮箱找回密码**功能。如忘记密码，请联系管理员重置，或通过已登录的其他设备到个人资料页修改。

## 5. 词库管理

点击导航栏 **"词库"** 进入。这是你管理单词的核心页面。

### 5.1 顶部统计概览

显示当前语言下：

- **整体掌握率**（绿色进度条）
- 4 个指标卡：词库容量 / 待复习 / 进行中 / 已掌握
- 7 阶段彩色柱状分布图

### 5.2 添加单词

左侧面板提供两种添加方式，通过顶部 Tab 切换。

#### 方式一：单个录入

1. 在输入框输入一个单词（如 `ephemeral`）。
2. 点击 **"添加"** 按钮。
3. 系统会调用 AI 自动生成：
   - 中文释义
   - 双语例句（含中文翻译）
   - 艾宾浩斯助记（词根 / 联想 / 谐音等记忆法）
   - 音标
   - 真人发音 URL
4. 成功后顶部出现绿色提示，输入框清空，可继续添加下一个。

> ⚠️ 若提示 **"单词已存在"**，说明该词已在你的词库中，不会重复添加。

#### 方式二：批量导入

支持两种输入方式：

- **拖拽 / 点击上传 `.txt` 文件**：点击虚线区域或直接把 txt 文件拖进去
- **直接粘贴文本**：在 textarea 中粘贴，每行一个词，或用逗号、分号、空格分隔

**生成模式选择（v1.9.3 新增）**

导入前可选择两种生成模式（卡片式单选，默认⚡快速）：

| 模式 | 适用场景 | 内部策略 | 30 词预估耗时 |
|---|---|---|---|
| **⚡ 快速**（默认） | 大部分用户，追求效率 | Gemini Flash Lite 主 + 字典 API 并行 + 并发 5 | ~6-9 秒 |
| **🎯 高质量** | 对释义/助记质量要求高 | GLM 5.2（实测质量最佳）+ 并发 5 | ~30 秒 |

- 选择会**自动记住**（localStorage），下次再导入时默认沿用上次的选择
- 两种模式互为兜底：主模型失败会自动切换到另一个模型，不会让导入失败
- 快速模式质量损失极小（9/10 vs 10/10），用户基本感知不到

操作步骤：

1. 输入内容后，文本框上方会显示 **"检测到 N 个"**
2. 选择生成模式（⚡ 或 🎯）
3. 点击 **"开始导入"**
4. 导入过程中显示 spinner 与 "parsing N / M"
5. 完成后：
   - 🟢 **成功**：列出所有新增词
   - 🟡 **跳过**：列出重复或失败的词及原因

> 💡 **建议**：单次批量导入控制在 30 个以内（硬上限）。v1.9.3 起后端并发 5 处理，30 词通常 6-30 秒完成。

### 5.3 浏览与筛选词汇

右侧列表区支持多种筛选：

- **状态 Tab**：全部 / 待复习 / 进行中 / 已掌握（含计数）
- **阶段筛选**：下拉选择阶段 0–6
- **排序**：最新 / 最旧 / 复习临近 / 字母顺序 / 阶段顺序
- **每页数量**：10 / 15 / 20 / 30 / 50
- **搜索框**：可匹配拼写、释义、例句、助记（不区分大小写）

底部提供分页器，支持上下翻页与跳转。

### 5.4 单词详情弹窗

点击任意词条打开详情弹窗，包含：

**头部：**

- 单词拼写（大字）
- 🔊 **发音按钮**：优先播放真人音频，失败时回退到浏览器语音合成
- 音标

**正文（4 个字段）：**

- **释义**
- **双语例句**（带边框块）
- **艾宾浩斯助记**（琥珀色块，带 Sparkles 图标）
- **学习参数**（4 格）：学习阶段 / 连胜记录（X/3）/ 基准日期 / 下次复习日期

**底部操作：**

- 🗑️ **删除**：彻底删除该词（会弹窗二次确认，操作不可逆）
- ✨ **AI 智能重新润色**：重新生成释义、例句、助记
- ✏️ **编辑详情**：切换到编辑模式，可手动修改音标、释义、例句、翻译、助记
- 💾 **保存修改** / **取消**（编辑模式下显示）

## 6. 复习会话

### 6.1 进入复习

有两种入口：

- 仪表盘点击 **"开始今日复习（N）"**
- 导航栏点击 **"复习"**（有待复习词时右上角显示红色徽章数字）

若没有待复习词，会显示空态卡：**"暂无待复习单词"**。

### 6.2 选择学习模式

开始会话前会让你三选一：

| 模式 | 图标 | 适用场景 |
|---|---|---|
| **闪卡模式** | FileText | 卡片翻转记忆，适合快速自测与浏览释义 |
| **拼写测试** | Compass | 键入拼写核对，深度建立字母与音节肌肉记忆 |
| **辨义选择** | CheckSquare | **看中文释义，从 6 个拼写相近的词里选对的**（v1.8 新增，适合攻克易混淆词） |

选择后点击 **"启动会话"**。

### 6.3 闪卡模式

**界面元素：**

- **顶部**：当前轮次徽章、错词重考数量徽章、进度（X/Y）
- **中央卡**：点击卡片任意区域可翻转
  - 背面：拼写 + 音标 + 🔊 发音按钮 + **"显示翻译"** 按钮
  - 正面：释义 + 双语例句 + 助记
- **底部决策按钮**（仅在翻面后出现）：
  - 🟢 **"记得"**（绿色）—— 记对了
  - 🔴 **"忘了"**（红色）—— 记错了

每张卡片出现时会**自动播放发音**。

### 6.4 拼写测试模式

**操作流程：**

1. 屏幕显示：音标 + 释义 + 挖空例句（拼写位置显示为 `首字母 + 下划线`，如 `e________`）
2. 在输入框键入你记忆中的拼写
3. 点击 **"检查"**（或回车）
4. 系统判定后：
   - ✅ 正确：输入框变绿，显示"拼写正确"
   - ❌ 错误：输入框变红，显示正确拼写供你对照
5. 点击 **"下一步"** 继续下一个词

> 💡 输入框已禁用浏览器自动补全和拼写检查，不会给你"提示"。

### 6.5 辨义选择模式

**核心玩法**：屏幕给出当前单词的**中文释义**作为题干，下方 6 个按钮排列 6 个拼写高度近似的英文单词（含正确答案 + 5 个干扰词），你需要选出哪个是正确的。

**操作流程：**

1. 启动会话后，系统会显示"正在准备复习材料..."（通常不到 1 秒）
2. 第一题出现：
   - 顶部：当前词的**中文释义**
   - 下方：**6 个拼写近似的英文单词**按钮（顺序已打乱）
3. 点击你认为是正确的那个单词
4. 系统立即判色反馈：
   - ✅ **你选对了**：你点的按钮变**绿色**
   - ❌ **你选错了**：你点的按钮变**红色**，同时**正确答案**的按钮变**绿色**高亮
5. 底部展开**所有 6 个单词的中文释义**，方便对照记忆（这是辨义模式独有的学习环节）
6. 点击 **"下一词"** 继续下一个

> 💡 **干扰词来源**：系统内置 10.4 万词的英汉词典 + 1.27 万词的中日词典，通过编辑距离算法实时找出拼写最相近的 5 个真实单词作为干扰项。**完全不依赖 AI**，所以速度极快（毫秒级）且永不重复/失败。英语词库走英汉词典，日语词库走中日词典，系统按当前词的语言**自动路由**。

> 💡 **为什么有时会看到"奇怪"的干扰词？** 对于复合词（如英语 `behind-the-meter` 或日语复合动词 `押し出す`），算法会取词组里最长的部分（英语）或去掉最后一个字符（日语）去找相似词，所以干扰项可能跟原词的主题不完全一致——这是正常的 fallback 行为。

**答错后的恢复选项**（极少数情况下，遇到非常生僻的词无法找到足够干扰词）：

- **重试**：重新生成干扰词
- **切回闪卡**：切到闪卡模式继续当前复习队列
- **退出会话**：返回仪表盘

### 6.6 错词循环重考（核心机制）

**规则：**

- 复习中**答错的词会被自动加入本会话的"重考队列"**
- 当本轮所有词过完后，若重考队列非空，会**自动开启新一轮**，只考刚才错的词
- 错词在新一轮中答对后，即从重考队列移除
- 直到重考队列清空，会话才算真正结束

**重要：** 最终的"首通正确率"统计只看**每词的第一次作答**，重考答对不影响首通数据。

### 6.7 会话完成

完成所有复习后显示结算页：

- 🏆 大奖牌图标
- **3 项核心数据**：总复习数 / 首通正确数 / 首通正确率（≥80% 高亮绿色）
- **AI 评语**：根据正确率动态生成鼓励或建议
- **首通错误词列表**（若有错词）：列出所有首通答错的词，便于回顾

## 8. 记忆算法原理

理解算法能帮助你更高效地学习。

### 8.1 6 阶段间隔重复

单词从新词到完全掌握，需要经过 6 个阶段，每阶段复习间隔如下：

```
新词 → 阶段 0 → 阶段 1 → 阶段 2 → 阶段 3 → 阶段 4 → 阶段 5 → 已掌握
       +1天      +2天      +4天     +7天     +15天    +30天
```

### 8.2 答对的奖励

每次**首轮答对**（first try correct）：

1. 连胜数 +1
2. **若连胜数 ≥ 3**：直接跃迁到阶段 5（下次复习 = 30 天后），进入快速掌握通道
3. **否则**：正常升一阶，按新阶段的间隔排下次复习

### 8.3 答错的惩罚

**首轮答错**会触发重置：

1. 阶段回到 0
2. 连胜数清零
3. 下次复习 = 明天（即第二天立即重练）

### 8.4 "已掌握"的判定

满足以下任一条件即视为已掌握：

- 阶段 ≥ 5（通过完整间隔升上来）
- 或连胜数 ≥ 3（通过快速通道）

已掌握的词不再排期复习，但你可以到词库手动管理它们。

### 8.5 错词循环 vs 首通正确率

- **错词循环**：会话内立即重考，确保你在本次复习中"记住"了它
- **首通正确率**：只统计每词的第一次作答结果，反映真实记忆水平

两者配合：错词循环保证短期记忆形成，首通正确率反映长期学习成效。

> 💡 这意味着你可以**完全从周计划出发管理学习**：周一计划「背 10 个葡语足球术语」，关联好单词，到时间点一下「▶ 一键复习」即可，复习完任务自动打勾。

### 11.9 多设备同步（v1.9.1）

- 所有计划、任务、任务类型配置都存在云端数据库
- 在手机创建的计划，电脑登录也能看到
- **老数据自动迁移**：如果你之前在 v1.9 初版用过周计划（存在浏览器本地），首次登录 v1.9.1 时会自动把本地数据上传到云端，迁移成功后清空本地缓存
- 迁移过程幂等：重复登录不会重复上传（按计划 ID 去重）

### 11.10 常见问题

**Q：为什么我的任务卡片的关联词消失了？**
A：可能是你从词库删除了那个单词。我们采用**懒清理**策略——删除单词时不会自动清理 `linked_word_ids` 数组，前端在渲染时过滤掉找不到的单词。如果你看到关联词数量变少，请检查词库。

**Q：我能在多个计划里关联同一个单词吗？**
A：可以。`linked_word_ids` 只存 ID，没有唯一约束，一个单词可以出现在任意多个任务里。

**Q：休息日会自动跳过进度计算吗？**
A：会。休息日不参与「本周打卡进度」的计算（分母仍为 7，但休息日的 `completed` 状态被强制为 false）。

## 12. 常见问题（FAQ）

### Q1：添加单词后 AI 释义生成失败怎么办？

**A：** 通常是 AI 服务临时不可用。你可以：

1. 稍后重试添加
2. 词库中点击该词详情，使用 **"AI 智能重新润色"** 重新生成
3. 也可以手动编辑详情填入自己的释义

### Q2：点击发音按钮没有声音？

**A：** 可能原因：

- 浏览器未允许自动播放音频 → 点击页面任意位置后再试
- 系统音量为 0 或静音
- 网络问题导致音频 URL 加载失败 → 系统会自动回退到浏览器 TTS 语音合成

### Q3：批量导入卡住不动？

**A：** v1.9.3 起后端已改为**并发 5 处理**，30 词通常 6-30 秒完成（取决于选择的生成模式）。如果你感觉特别慢：
- 确认是否选了 **🎯 高质量模式**（GLM 5.2，~30 秒）
- 网络不稳定时 Gemini 会回退到 GLM，会稍慢
- 单次上限固定为 30 词，超过会自动截断

### Q4：忘记密码怎么办？

**A：** 应用暂未提供邮箱找回密码。请联系管理员协助重置，或通过其他已登录设备到个人资料页修改。

### Q5：连续复习天数是怎么计算的？

**A：** 系统按你**每天是否有复习记录**来计算连续天数。某天没有复习记录，连续天数会中断。建议每天至少完成一次复习会话以保持连胜。

### Q6：切换目标语言后，之前的词库会丢失吗？

**A：** 不会。所有词库都按语言分类存储，切换语言只是筛选显示。切回原来的语言即可看到之前的所有单词。

### Q7：已掌握的词还能再复习吗？

**A：** 系统默认不再为已掌握的词排期。但你可以到词库中点击该词查看详情，或删除后重新添加来重新学习。

### Q8：管理员功能（时光机 / 数据库重置）普通用户为什么看不到？

**A：** 这两个功能属于高危操作，仅对管理员邮箱开放。普通用户仪表盘不会显示这两块卡片，避免误操作带来数据风险。如果你确实需要这些功能，请联系管理员。

### Q9：周计划数据会同步到云端吗？

**A：** v1.9.1 起所有周计划数据（计划 / 任务 / 任务类型）都存在云端数据库，多设备同步。如果你之前用过 v1.9 初版（数据在浏览器本地），首次登录 v1.9.1 时会自动迁移到云端。详见 §11.9。

### Q10：我误删了一个周计划，能恢复吗？

**A：** 目前不行。删除计划是硬删除（CASCADE 删除所有任务和日级元数据）。建议删除前用归档功能（左侧面板的归档按钮）先把计划收起，确认不再需要再删除。

## 13. 开发者协作流程（Git Flow）

> 本节面向参与本项目开发的工程师 / AI Agent，普通用户无需关心。

### 12.1 分支策略

本项目遵循标准 Git Flow：

| 分支 | 用途 |
|---|---|
| `main` | 生产分支，禁止直接提交。每次合并自动触发 Vercel 部署 |
| `develop` | 开发集成分支（预留，目前主要在 main 上集成）|
| `feature/<描述>` | 新功能分支，从 `main` 拉出，完成后合回 |
| `hotfix/<描述>` | 紧急修复分支，从 `main` 拉出，合回 `main` |

### 12.2 提交规范（Angular）

格式：`<type>(<scope>): <subject>`

- **type**：`feat` / `fix` / `docs` / `style` / `refactor` / `test` / `chore`
- **scope**：`auth` / `word-list` / `review` / `learning-plans` / `dashboard` / `navbar` / `i18n` / `db` / `api` / `e2e`
- **subject**：一句话简洁描述（中文）

示例：`feat(learning-plans): 引入周计划学习日程系统并落到数据库`

### 12.3 完整流程（6 步 SOP）

1. 检查环境：`git status && git branch -vv`
2. 同步主干：`git checkout main && git pull origin main`
3. 拉特性分支：`git checkout -b feature/<描述>`
4. 开发 + 提交：按文件名逐个 `git add`（不要用 `-A`/`.`），用 `git commit -F` 避免转义问题
5. 推送：`git push -u origin feature/<分支>`
6. 合并 main（**需用户确认**）：`git checkout main && git merge --no-ff feature/<分支>` → `git push origin main` → 清理分支

### 12.4 关键红线

- ❌ 绝不在 `main` 上直接 commit
- ❌ 绝不 force push 到 main / develop
- ❌ 绝不未经用户确认就合并 main
- ❌ 绝不用 `git add .` / `-A`（容易带入 `.env`）
- ❌ 绝不修改 `git config` 全局配置
