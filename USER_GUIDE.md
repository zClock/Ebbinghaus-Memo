## 快速开始

### 1. 注册账号

1. 打开 <https://ebbinghaus-memo.vercel.app/>
2. 在登录卡片点击 **「注册」** 切换到注册视图
3. 填写邮箱、密码、昵称、**每日目标（新学单词数）**
4. 提交后自动登录，进入空 Dashboard

### 2. 初始状态

- 词库默认为空
- UI 默认显示**中文**
- 目标词库默认为**英语**
- 所有单词需要用户主动添加（应用不预置种子数据）

## 背单词核心流程

整体节奏：**添加单词 → 按时复习 → 错词重考 → 持续巩固**

### 第 1 步：添加单词

进入「词库」标签页 → 点击「添加单词」或「批量导入」。

### 第 2 步：AI 自动生成释义

添加单词后，应用会调用 AI（Gemini → GLM 兜底）自动生成：
- **音标**（phonetic）
- **中文释义**（definition）
- **英文例句 + 中译**（example / exampleTranslation）
- **助记法**（mnemonic）

对于英语单词，还会调用 dictionaryapi.dev 做数据增强。

### 第 3 步：按算法复习

应用使用 **6 阶段 SRS（间隔重复）算法**：

| 阶段 | 第几天复习 |
|---|---|
| 1 | 第 1 天 |
| 2 | 第 2 天 |
| 3 | 第 4 天 |
| 4 | 第 7 天 |
| 5 | 第 15 天 |
| 6 | 第 30 天 |

**判定规则**：
- 答对 → 进入下一阶段
- 答错 → 回到阶段 1
- **连续答对 3 次 或 阶段 ≥5** → 标记为「已掌握」

### 第 4 步：错词重考

复习中答错的单词会进入**错词队列**，在本轮复习结束后自动重考（复习队列逻辑由独立的纯函数 `reviewQueue.ts` 实现）。

## 词库管理

### 单个添加

1. 「词库」→「添加单词」
2. 输入单词 → 提交
3. AI 生成释义（约 2-5 秒）
4. 列表实时刷新

### 批量导入

1. 「词库」→「批量导入」
2. 文本框中粘贴多个单词（每行一个，最多 30 个）
3. 点击导入，进度条实时显示
4. AI 会**并发**生成释义，期间可看到进度

### 编辑与删除

- 点击列表中单词 → 编辑详情（音标、释义、例句、助记）
- 删除按钮 → 确认后移除

### AI 重生成释义

如果某条单词的释义质量不佳，点击「AI 重新生成」按钮，系统会用 AI 重写一遍（传入当前 `language` 参数，保证多语言适配）。

### 搜索与过滤

- 顶部搜索框：全文匹配（单词、释义、例句）
- 语言 Tab 切换：按目标语言过滤（English / Japanese / Spanish / French / Portuguese / All）

## 复习会话

### 开始复习

Dashboard 首页「今日复习」卡片显示：
- 今日待复习数量
- 已掌握数量
- 连续打卡天数

点击「开始复习」进入会话。

### 闪卡模式

- 展示单词 + 音标 + 例句（挖空）
- 点击「显示答案」翻面
- 自评「认识 / 不认识」

### 拼写测试

对部分单词，会要求拼写答案（Unicode-safe，支持日/西/法/葡等非英语字符）。

### 错词重考

一轮复习结束后，所有答错的单词会进入重考队列，循环直到全部答对或达到上限。

### 历史记录

每次复习会记录到 `histories` 表，Profile 页可查看总复习次数、平均正确率。

## 多语言切换

### UI 语言（6 种）

应用界面支持 **6 种 UI 语言**：

| 代码 | 语言 | 状态 |
|---|---|---|
| zh | 中文 | ✅ 默认 |
| en | English | ✅ |
| ja | 日本語 | ✅ |
| es | Español | ✅ |
| fr | Français | ✅ |
| pt | Português | ✅ |

**切换方法**：Dashboard 右上角「设置」图标 → 选择 UI 语言。所有按钮、标签、提示立即跟随。

### 目标词库（5 种）

你要背的语言（词库内容）支持：

- English / Japanese / Spanish / French / Portuguese

每个词库独立管理，互不干扰。

### 特殊说明：缺失键兜底

某些翻译键可能尚未定义，此时会自动回退到 **中文兜底**（不会显示空白）。

## 常见问题（FAQ）

### Q: 添加单词时 AI 生成释义失败？

**A**: 应用使用 Gemini → GLM 双兜底。如果两个都失败：
- 检查网络
- 稍后重试
- 点击「AI 重新生成」按钮

### Q: 忘记密码怎么办？

**A**: 当前版本**没有**找回密码功能。联系管理员（如果是开发环境）或直接查数据库重置。

### Q: 词库可以批量导出吗？

**A**: 当前版本不支持导出。如需迁移，可直接读取 Supabase Postgres 或本地 `data/db.json`。

### Q: 时间旅行会影响其他用户吗？

**A**: **会**。时间偏移 `system_offset_ms` 是全局的，所有用户共享。未来计划改为按用户隔离（见 plan.md §4.1）。

### Q: 为什么日/西/法/葡的足球规则正文显示英文？

**A**: 因为 FIFA 规则数据本身只有中英双语版本。UI 控件已完整本地化，但规则正文未翻译。中文 UI 用户可以看到中英对照。

### Q: 如何部署自己的实例？

**A**: 项目支持 Vercel 部署。参考 [CLAUDE.md](./CLAUDE.md) 的技术栈章节，需要配置以下环境变量：
- `GEMINI_API_KEY`（必填）
- `SUPABASE_URL` / `SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY`（切到云端数据库）
- `GLM_API_KEY` / `GLM_BASE_URL` / `GLM_MODEL`（可选，AI 兜底）

### Q: 数据存储在哪里？

**A**:
- **生产（Vercel）**：Supabase Postgres
- **本地开发**：`data/db.json`（默认，不入库）
- **E2E 测试**：`data/db.test.json`（独立，不污染开发数据）
